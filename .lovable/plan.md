

## Переработка мобильного каталога -- L1 и L2 всегда раскрыты, "Показать ещё" только для L3

### Что будет сделано

Полная переработка `MobileCatalogMenu.tsx`:
- Все 3 уровня видны сразу на одном экране, без картинок
- Первый уровень (категории) -- всегда показаны все
- Второй уровень (подкатегории) -- тоже всегда показаны все
- Третий уровень (товарные группы) -- показаны первые 3, остальные за ссылкой "Показать ещё"
- Справа от каждого пункта -- круглая радио-галочка, выровненная по правому краю
- Отступы по уровням: `pl-4`, `pl-8`, `pl-12`

### Технические детали

**Файл: `src/components/MobileCatalogMenu.tsx`** -- полная переработка:

1. **3-уровневая структура данных** без `imageUrl`:
   - `CategoryL3`: `{ id, name }`
   - `CategoryL2`: `{ id, name, children: CategoryL3[] }`
   - `CategoryL1`: `{ id, name, children: CategoryL2[] }`

2. **Состояния**:
   - `expandedL3: Set<string>` -- какие L2-подкатегории показывают все свои L3-пункты (по умолчанию только первые 3)
   - `selectedId: string | null` -- выбранный пункт для радио-галочки

3. **Рендеринг**:
   - Все L1 отображаются всегда, полностью
   - Все L2 под каждым L1 отображаются всегда, полностью
   - L3 под каждым L2 -- первые 3 видны, остальные за "Показать ещё" (с отступом `pl-12`). Клик добавляет id L2-родителя в `expandedL3`

4. **Каждая строка пункта**:
   - Flex-контейнер: текст слева с отступом по уровню, радио-галочка справа через `ml-auto mr-4`
   - Выбранный пункт: заполненный кружок с иконкой Check

5. **Строка "Показать ещё"**:
   - Текстовая ссылка `text-sm text-primary` с отступом `pl-12` (уровень L3)
   - Без галочки справа
   - Появляется только если L3-пунктов больше 3
