

## Эмодзи и файл при написании комментария

### Что будет сделано

В области ввода комментария (нижняя панель попапа) добавить две кнопки слева от поля ввода:

1. **Кнопка эмодзи** (иконка `Smile`) -- открывает Popover с сеткой популярных эмодзи. Клик по эмодзи вставляет его в текст комментария.
2. **Кнопка прикрепить файл** (иконка `Paperclip`) -- открывает диалог выбора файла (изображения). Выбранный файл показывается как превью над полем ввода с кнопкой удаления.

### Визуальная структура

```text
┌──────────────────────────────────────┐
│ ... комментарии ...                  │
├──────────────────────────────────────┤
│ [превью файла]                [x]    │  <- только если файл выбран
├──────────────────────────────────────┤
│ [😊] [📎]  [Написать комментарий...] [➤] │
└──────────────────────────────────────┘
```

### Технические детали

**Файл: `src/components/PostCard.tsx`**

1. Импортировать `Smile`, `Paperclip`, `X` из `lucide-react`
2. Импортировать `Popover, PopoverTrigger, PopoverContent` из `@/components/ui/popover`
3. Добавить состояние `attachedFile: { file: File; preview: string } | null`
4. Создать массив популярных эмодзи (20-30 штук: улыбки, жесты, сердечки, и т.д.)
5. Компонент EmojiPicker внутри Popover -- сетка `grid-cols-8` с кнопками-эмодзи. Клик вставляет эмодзи в `newComment` и закрывает попап.
6. Скрытый `<input type="file" accept="image/*">` + кнопка `Paperclip` для его триггера
7. При выборе файла -- создать `URL.createObjectURL` для превью, сохранить в стейт
8. Показать превью файла над полем ввода (маленькая картинка 48x48 с кнопкой X для удаления)
9. При отправке комментария -- очистить и файл тоже (сам файл пока не загружается на сервер, просто UI)
10. Обновить интерфейс `PostComment` -- добавить опциональное поле `attachment?: string` для URL превью
